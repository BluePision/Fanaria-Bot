## 設計思想
- JSONファイルをデータベースとして使用しない
  > JSONは
  > 書き換え中に中断してしまうとファイル全体が壊れやすく、排他制御やトランザクションがない ため、  
  > 排他制御やトランザクションが使え、大量のデータ管理や検索にも強いデータベース SQLite を使用する

- データベースはSQLite（`sqlite3`）で使用する
  > 恐らく一番運用が楽なため

- 機能ごとにファイルやフォルダを可能な限り細分化する
  > 主に可読性を確保するため

- コマンドは同系統のコマンドをGroupコマンド（`discord.app_commands.Group`）としてまとめて登録する
  > 主に見栄えのため

- コマンドに限らず全機能はFanaria内でのみ使用できるように徹底する
  > 公式サーバーBotとして他のサーバーで使えてしまうのはおかしいため

- 基本的にIDや各種設定値は`bot/configs/`内で管理する
  > Let's aim for code without magic numbers.

- 関数を大量に使ってゴリ押さず、クラスを使う
  > 見栄えもいい、可読性もいい、管理もしやすいため

- 所々に遊びを含めて楽しませる
  > 厳格よりは楽しげじゃん？

- このBotからすると管理者達はおもちゃです
  > 管理者だからね、しょうがないね

- 音楽、特にEDMを好むBotです
